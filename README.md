Цикломатска комплексност
Цикломатската комплексност на двете функции ја пресметувам според формулата V(G) = P + 1, каде што P е бројот на предикатни јазли (точки на одлучување каде што патот се дели на два).

1.Пресметка за searchBookByTitle
Врз основа на кодот, предикатните јазли се:

if (title.isEmpty()) — Проверка за влезен параметар.
for (Book book : books) — Услов за продолжување на циклусот.
if (book.getTitle().equalsIgnoreCase(title) && !book.isBorrowed()) — Сложен услов за филтрирање.
if (results.isEmpty()) — Проверка за враќање на null.

Пресметка V(G) = 4 + 1 = 5

2.Пресметка за borrowBook
if (title.isEmpty() || author.isEmpty()) — Проверка на влезните стрингови.
for (Book book : books) — Услов за пребарување низ листата.
if (book.getTitle().equalsIgnoreCase(title) && book.getAuthor().equalsIgnoreCase(author)) — Проверка за точна книга.
if (!book.isBorrowed()) — Проверка на достапноста.

Пресметка V(G) = 4 + 1 = 5

Цикломатската комплексност од 5 индицира дека постојат 5 линеарно независни патеки низ кодот. Овој број ни служи како водич за тоа колку минимум тест случаи се потребни за да се постигне целосно покривање на патеките (path coverage).

borrowBook
Прв случај (T/X): Внесуваме празен наслов (title). Тука програмата веднаш фрла исклучок без да го гледа авторот.
Втор случај (F/T): Внесуваме валиден наслов, но празен автор. Програмата мора да го провери вториот дел од условот за да го фрли исклучокот.
Трет случај (F/F): Внесуваме и наслов и автор. Ова е единствениот пат кога изразот е лажен и програмата продолжува со извршување на логиката за позајмување.

searchBookByTitle
Прв случај (T/T): Внесуваме наслов кој постои и книга која не е позајмена. Ова е единствената патека каде книгата успешно се враќа во резултатите.
Втор случај (T/F): Насловот се поклопува, но книгата е веќе позајмена. Овој тест е клучен за да се потврди дека системот правилно ја филтрира достапноста.
Трет случај (F/X): Внесуваме наслов кој воопшто не постои во листата. Тука Java веднаш прекинува со проверка на тој објект бидејќи првиот услов не е исполнет.

Минимален број на тест случаи за ова функција според Multiple condition критериумот e 3.
